Comparison between pure Java Web Start and Ice Boar

Web Start Ice Boar Description
helloworld-simple.jnlp helloworld-simple-ice-boar.jnlp This example show a minimum example of both runners. Starting one jar without dependencies
helloworld-dependencies.jnlp helloworld-dependencies-ice-boar.jnlp Starting one jar with dependencies
helloworld-1.7.0_04.jnlp helloworld-1.7.0_04-ice-boar.jnlp Run on exactly JRE 1.7.0_04 version. Pure Web Start will be probably started not on 1.7.0_04
helloworld-run-on-user-jvm.jnlp helloworld-run-on-user-jvm-ice-boar.jnlp Run on user current used JVM >=1.5 (pre-installed). A behaviour of pure Web Start is not determinate - it deepens on user installed JVM (Since 0.8)
helloworld-heap-size.jnlp helloworld-heap-size-ice-boar.jnlp Define heap size (initial and max)
helloworld-vm-args.jnlp helloworld-vm-args-ice-boar.jnlp Define VM arguments
helloworld-app-args.jnlp helloworld-app-args-ice-boar.jnlp Define application arguments

Ice Boar specific features

Ice Boar Description
helloworld-show-debug-console-ice-boar.jnlp Show debug console
helloworld-not-close-on-end-ice-boar.jnlp Don't close Ice Boar on end - for debugging purposes
helloworld-frame-title-ice-boar.jnlp Change Ice Boar frame title to "Custom Frame Title"
helloworld-icon-ice-boar.jnlp Change Ice Boar Icon. Icon of progress frame will be red. Another icons without changes. (Since 0.6)
helloworld-splash-ice-boar.jnlp Using splash 1 (Since 0.7)
helloworld-undecorated-ice-boar.jnlp Using splash 2 and hide frame border (undecorated) (Since 0.7)
helloworld-force-not-use-user-jvm-ice-boar.jnlp Force to always use target (downloaded or from cache) JVM (Since 0.8)
